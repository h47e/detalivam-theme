# Roadmap админки ДеталиВам

Цель: сделать админку темы единым рабочим инструментом для витрины, WooCommerce, SEO, файлов и обслуживания без правки PHP-кода на каждую мелкую настройку.

## Текущий статус

- Фаза 1: в работе, общий каркас, токены статусов, футер и быстрые действия введены.
- Фаза 2: в работе, `Ctrl K` расширен сервисными командами и безопасными POST-действиями, `/` фокусирует видимый поиск на страницах админки.
- Фаза 3: в работе, dashboard показывает состояние товаров, SEO, профиля, Ozon, uploads, резерва, релиза и сервисных действий.
- Фаза 4: в работе, добавлен журнал сервисных операций с экспортом и очисткой, рискованные uploads/backup/cache/search-кнопки используют общий слой подтверждения.
- Фаза 5: в работе, dashboard и тяжелые сводки используют кэш/сохраненные результаты, диагностика переиспользует один контекст отчетов, uploads не сканируется при каждом открытии.
- Фаза 6: в работе, палитра `Ctrl K` получила `aria-expanded`, `aria-controls` и активный пункт; отмена рискованных действий озвучивается через `aria-live`.
- Фаза 7: в работе, smoke-check покрывает ключевые PHP-файлы, страницы, формы команд и `admin-post` действия.

## Фаза 2. Навигация и Ctrl K

- [x] Палитра `Ctrl K` ищет страницы, быстрые действия и сервисные команды.
- [x] Команды ведут к безопасным экранам: диагностика, аудит uploads, очистка uploads, SEO-задачи, резерв, slug, индекс поиска.
- [x] Палитра запускает безопасные POST-действия: пересборка live-search и очистка служебных кэшей.
- [x] Локальный поиск `/` фокусирует видимый поиск на текущей странице.
- [ ] Добавить команды для прямого перехода к проблемным блокам из dashboard-задач, если список задач расширится.
- Критерий готовности: администратор попадает к любой ключевой операции за 1-2 действия.

## Фаза 3. Dashboard

- [x] Один экран состояния: товары, SEO, профиль магазина, Ozon, uploads, резерв, обслуживание, релиз и последние изменения.
- [x] Понятные статусы `OK / проверить / ожидает` с текстовыми подсказками.

## Фаза 4. Безопасные операции

- [x] Рискованные backup/cache/search/uploads-действия используют подтверждение.
- [x] Удаление uploads остается переносом в backup, не физическим удалением.
- [x] Журнал действий хранит: кто, когда, что сделал, сколько объектов затронуто.
- [x] Журнал поддерживает экспорт и очистку с подтверждением.
- [x] Пересборка live-search записывается в сервисный журнал.
- [ ] Добавить единый экран отката по последним backup/uploads-действиям, если понадобится быстрый rollback из одного места.
- Критерий готовности: массовое действие можно объяснить, проверить и откатить.

## Фаза 5. Производительность

- [x] Тяжелые dashboard-сводки кэшируются.
- [x] Диагностика переиспользует один контекст отчетов вместо повторных пересчетов.
- [x] Uploads-аудит не сканирует файлы при каждом открытии страницы.
- [x] Большие uploads-списки показываются превью, CSV и фильтрами, без рендера тысяч строк.
- [x] Live-search имеет ручную пересборку индекса из настроек и `Ctrl K`.
- [x] Добавлена индикация возраста live-search индекса в диагностике обслуживания.
- [x] Добавлена индикация возраста dashboard cache, аудита товаров и uploads-аудита без тяжелых пересчетов.
- Критерий готовности: админ-страницы открываются быстро даже при росте каталога и uploads.
