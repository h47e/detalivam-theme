# Семантический план (волна 1, 36 запросов)

Цель: закрыть первые 30–50 высоковероятных запросов по запчастям ВАЗ/технике с явным
размесчением по посадочным страницам (category / subcategory / product / landing).

Критерии выбора посадки:
- **category**: общий запрос по группе товаров, нужен общий список/каталог.
- **subcategory**: конкретная группа внутри категории (например, для определённой модели/позиции).
- **landing**: сравнительные/информационные запросы, где нужна страница с текстом и подборкой карточек.
- **product**: точечные запросы по артикулу.

## Шаблон обработки
- **Страница**: ссылка на целевую посадку (slug / URL будет точнее после аудита структуры).
- **Интент**: коммерческий (покупка/цена/наличие) или информационный (подбор/совместимость).
- **Цель**: заказ/звонок/поиск/клик в Ozon.

## Как переводить в исполнение (потом в редактор/админку)
1. Для каждой позиции проставить итоговый URL из структуры:
   - `/catalog/` — общая категория;
   - `/catalog/{category}` — подкатегория;
   - `/product/{slug}` — товар;
   - `/landing/{slug}` — информационный блок.
2. Сразу в карточку добавить:
   - в title: ключ + модель + бренд + «купить/цена/наличие»,
   - в description: один call-to-action на действие,
   - 1-2 FAQ по теме запроса.
3. Для блоков с «наличие» и «цена» в выдаче акцент делать в meta-блоках карточек и выдачи.
