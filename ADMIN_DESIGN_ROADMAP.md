# Roadmap дизайна админки ДеталиВам

Цель: сделать админку темы более спокойной, плотной и удобной для ежедневной работы: меньше декоративного шума, больше читаемости, единый ритм карточек, форм, статусов и навигации.

## Текущий статус

- Фаза 1: готово, общая визуальная основа, header/nav и проверяемые дизайн-токены введены.
- Фаза 2: готово, общий слой плотности для карточек, форм, таблиц и кнопок добавлен.
- Фаза 3: готово, базовая система статусов, бейджей, счетчиков и опасных действий унифицирована.
- Фаза 4: в работе, общий ритм сложных страниц закреплен, точечная визуальная полировка остается для живого прохода в админке.
- Фаза 5: в работе, мобильные и keyboard-правила добавлены на уровне общего CSS, живой проход на реальных экранах еще нужен.
- Фаза 6: готово для текущего прохода, visual audit расширен, smoke-check и diff-check пройдены.

## Фаза 1. Визуальная Основа

- [x] Создать отдельную дорожную карту дизайна админки.
- [x] Уплотнить общий header: меньше декоративного градиента, спокойнее фон, ниже высота.
- [x] Упростить верхние tabs и быстрые действия: ровнее сетка, меньше визуального шума.
- [x] Добавить дизайн-токены для surface, border, shadow, radius и закрепить их visual audit.
- Критерий готовности: все страницы сразу выглядят спокойнее, а общий стиль контролируется проверкой.

## Фаза 2. Рабочая Плотность

- [x] Привести крупные карточки к единому радиусу, тени и внутренним отступам.
- [x] Уменьшить лишние вертикальные отступы в формах без потери читаемости.
- [x] Унифицировать таблицы и списки: одинаковые строки, hover/focus, пустые состояния.
- [x] Проверить, что важные кнопки не теряются среди вторичных.
- Критерий готовности: на экране помещается больше полезных настроек без ощущения тесноты.

## Фаза 3. Статусы И Акценты

- [x] Ввести единую шкалу статусов: OK, проверить, пусто, опасно.
- [x] Снизить доминирование синего там, где это не действие и не активный пункт.
- [x] Привести бейджи, счетчики и progress-блоки к одной форме.
- [x] Сделать опасные действия визуально отличимыми от обычных сервисных.
- Критерий готовности: статус считывается текстом, формой и цветом, а не только оттенком синего.

## Фаза 4. Сложные Страницы

- [x] SEO: сделать очередь задач и технические блоки плотнее и легче для сканирования.

- [x] Файлы: привести audit/cleanup/backup таблицы к общему ритму.
- [x] Магазин: выровнять профильные формы, logo/media блоки и marketplace секции.

- Критерий готовности: каждая тяжелая страница похожа на часть одного инструмента, а не на отдельный экран.
