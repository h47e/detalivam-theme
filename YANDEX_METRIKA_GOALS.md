# Яндекс.Метрика: цели для трафика и продаж

В теме добавлен базовый слой отправки целей. Он работает так:
- если задан ID счётчика, вызывается `ym(counterId, 'reachGoal', goalName, params)`;
- параллельно событие уходит в `window.dataLayer`;
- также создаётся браузерное событие `dv:metrika-goal` для отладки.

ID счётчика задаётся в админке: `ДеталиВам` → `Настройки` → `Аналитика` → `ID счётчика Яндекс.Метрики`.
Технически значение хранится в `dv_theme_options[yandex_metrika_counter_id]`.

Статус можно проверить в двух местах:
- `Консоль WordPress` → виджет `ДеталиВам: состояние магазина` → карточка `Analytics`;
- `ДеталиВам` → `Настройки` → `Диагностика` → карточка `Яндекс.Метрика`.

Важно: тема не выводит код счётчика и не создаёт второй счётчик. Если счётчик уже подключён плагином Яндекс.Метрики, это нормально: тема только отправляет цели в уже существующий `ym(...)`.

## Цели, которые нужно создать в Метрике
| Цель | Название в Метрике | Когда срабатывает | Главная польза |
|---|---|---|---|
| `add_to_cart` | Добавление в корзину | Успешный AJAX add-to-cart | Понимать, какие страницы реально продают |
| `phone_click` | Клик по телефону | Любой клик по `tel:` | Отдельно считать звонки |
| `site_search` | Поиск по сайту | Отправка формы поиска | Видеть спрос, которого не хватает в структуре |
| `ozon_click` | Переход на Ozon | Клик по Ozon-ссылке/кнопке | Оценить внешний канал продаж |
| `product_click` | Клик в карточку товара | Клик по товару из выдачи/подборки | Понять CTR посадочных |
| `checkout_start` | Начало оформления заказа | Переход в checkout | Видеть прогрев до заказа |
| `order_success` | Заказ оформлен | Страница `order-received` | Главная e-commerce конверсия |

## Как проверить
1. В админке открыть `ДеталиВам` → `Настройки` → `Аналитика`.
2. Нажать `Открыть сайт в режиме проверки целей`.
3. На витрине выполнить реальные действия: добавить товар в корзину, нажать телефон, поиск, Ozon, checkout.
4. Убедиться, что в правом нижнем углу панели появляются события целей.

Дополнительно можно проверить через консоль:
   ```js
   window.addEventListener('dv:metrika-goal', (event) => console.log(event.detail));
   ```

Debug-панель включается только для администратора и только при параметре `?dv_metrika_debug=1`.

## Как связать с первой волной семантики
Для файла `TRAFFIC_SEMANTICS_BATCH1_WORKBOARD.md` используем эти же имена целей:
- коммерческие подкатегории: `add_to_cart`, `phone_click`;
- запросы с ценой/наличием: `product_click`, затем `add_to_cart`;
- статьи/лендинги под подбор: `site_search`, затем `add_to_cart`;
- внешний канал: `ozon_click`;
- финальная продажа: `checkout_start`, `order_success`.
